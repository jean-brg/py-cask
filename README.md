# Cask
A minimalistic tool to compile Flask apps into a standalone application. 

Cask wraps your Flask app in a [pywebview](https://pywebview.flowrl.com) window and provides a simple CLI to handle the build process

## Requirements
- Python 3.11+
- [PyInstaller](https://pyinstaller.org) (for building executables)

## Installation
Install directly from GitHub:
```bash
pip install git+https://github.com/jean-brg/py-cask.git
```

To build executables, also install PyInstaller:
```bash
pip install pyinstaller
```

## Documentation
Full API reference is available in [/docs](/docs).

## Quickstart
Cask is a drop-in Flask replacement. Just swap your import and change your app object. All existing Flask imports work out-of-the-box, just from `cask` instead.

```python
# Before
from flask import Flask, render_template
app = Flask(__name__)

# After
from cask import Cask, render_template
app = Cask(__name__, app_name="My App")
```

Then call `run_as_app()` instead of `run()`:
```python
if __name__ == "__main__":
    app.run_as_app()
```

Your routes, templates, and Flask code work as-is. pywebview's menu API (`Menu`, `MenuAction`, `MenuSeparator`) is also importable directly from `cask`.

Run with:
```bash
python main.py
```
or
```bash
cask run
```

## Project Structure
Cask expects a standard Flask layout:
```
my-app/
├── main.py
├── static/
│   └── caskicon.icns   # or .ico on Windows/Linux (Optional)
├── templates/
│   └── index.html
└── instance/           # For runtime data (databases, config, etc.) (Optional)
```

## Packaging into an Executable
```bash
cask init    # scans your project and generates cask.toml
cask build   # packages your app with PyInstaller
```

Your built app will be in `./dist/<app name>`.

### cask.toml
```toml
# Generated by cask init
[app]
name = "My App"
entry = "main.py"
icon = "static/caskicon.ico"

[data]
"static" = "static"
"templates" = "templates"
```

| Field       | Required | Description                                                          |
|-------------|----------|----------------------------------------------------------------------|
| `app.name`  | Yes      | Display name and executable name.                                    |
| `app.entry` | Yes      | Path to your Python entry file.                                      |
| `app.icon`  | No       | Path to a `.ico` (Windows/Linux) or `.icns` (macOS) icon file.       |
| `[data]`    | No       | Files and folders to bundle, as `"source" = "destination"` pairs.    |

Use `cask init -i` for interactive prompts instead of auto-discovered values.
